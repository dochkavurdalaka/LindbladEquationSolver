# Lindblad Equation Solver

![C++20](https://img.shields.io/badge/C++-20-blue.svg)
![Intel MKL](https://img.shields.io/badge/Intel%20MKL-Optimized-brightgreen)
![CMake](https://img.shields.io/badge/CMake-3.15+-lightgrey)

Высокопроизводительная библиотека на C++ для численного моделирования динамики открытых квантовых систем. Программа решает **марковское управляющее уравнение Линдблада (Lindblad master equation)** с использованием библиотеки **Intel MKL** (Math Kernel Library).

Главная особенность проекта — векторизация матрицы плотности путем разложения по ортогональному базису $SU(N)$. Такое представление позволяет гарантировать инвариантность существенных свойств матрицы плотности (эрмитовость и условие нормировки $\mathrm{Tr}(\rho) = 1$) на всех этапах численного интегрирования.

## 🔬 Физико-математическая модель

Стандартное уравнение Линдблада имеет вид:

$$ \dot{\rho}(t) = -i[H(t), \rho(t)] + \sum_{p} \gamma_p \left( L_p \rho(t) L_p^\dagger - \frac{1}{2}\{L_p^\dagger L_p, \rho(t)\} \right) $$

Прямое интегрирование (например, методом Рунге-Кутты 4-го порядка) не гарантирует сохранение ключевых физичесиких свойств матрицы плотности квантовой системы, таких как эрмитовость, условие нормировки $\mathrm{Tr}(\rho) = 1$ и положительная полуопределенность.

В данном проекте используется переход в **пространство Лиувилля**. Матрица плотности $\rho$, Гамильтониан $H$ и операторы Линдблада $L$ раскладываются по базису генераторов SU(N) (обобщенным матрицам Гелл-Манна $F_i$):

$$ \rho(t) = \frac{1}{N}I + \sum_{i=1}^{N^2-1} v_i(t) F_i $$

Это превращает комплекснозначное дифференциальное уравнение для матриц в вещественнозначную систему линейных дифференциальных уравнений для вектора $v(t)$:

$$ \dot{v}(t) = \left( Q(t) + R \right)v(t) + K$$

Где:
*   $Q(t)$ — матрица, отвечающая за унитарную эволюцию (зависит от Гамильтониана).
*   $R$ — матрица диссипации (зависит от матрицы Коссаковски).
*   $K$ — вектор сдвига.


## 🚀 Основные возможности

*   **Генерация SU(N) базиса и тензоров:** Эффективный расчет тензоров $f_{mns}$ (коммутаторы) и $d_{mns}$ (антикоммутаторы).
*   **Сборка разреженных матриц:** Быстрое конструирование матриц $Q$ и $R$ для специальной физической модели димера в формате CSR (Compressed Sparse Row) средствами Intel MKL.
*   **Интегратор RK4:** Классический метод Рунге-Кутты 4-го порядка, оптимизированный под `cblas_dgemv` и `mkl_sparse_d_mv`.
*   **Димерные модели (Dimer Model):** Встроенная поддержка систем с зависящим от времени Гамильтонианом $H(t)$ (например, ступенчатая функция $\theta{(t)}$). Модуль `SparseQBuilder` позволяет мгновенно обновлять ненулевые элементы матрицы $Q(t)$ без переаллокации памяти.
*   **Бенчмарки и тесты:** Набор тестов для проверки корректности математики (совпадение быстрых тензорных и матричных расчетов с медленными "в лоб") и замеров производительности.

    ├── main.cpp                # Интегрирование вещественнозначной системы, быстрое вычисление ключевых структурных компонент
    ├── main_dimer.cpp          # Пример с димером (зависящий от времени разреженный H, независящий от времени разреженный L), оптимизированная реализация

## 📂 Подробная структура проекта

Ниже представлено детальное описание архитектуры кода, разделенное на заголовочные файлы и исполняемые файлы с логикой симуляции.

### 📁 Директория `/include` (Заголовочные файлы)

*   **`mkl_complex16.h`**
    Вспомогательный файл, определяющий набор перегруженных операторов для типа `MKL_Complex16` (комплексные числа двойной точности из Intel MKL). Это позволяет использовать стандартные интуитивные арифметические операции (`+`, `-`, `*`, `/`) для данного типа.

*   **`generate_matrices.h`**
    Содержит функции для генерации случайных матриц, используемых для инициализации симуляции:
    *   `GenerateDensity`: Создает физически корректную случайную матрицу плотности $\rho$ путем генерации случайной матрицы $A$, вычисления $M = A A^\dagger$ и последующей нормировки $\rho = M / \mathrm{Tr}(M)$.
    *   `GenerateTracelessHamiltonian`: Генерирует случайный эрмитов гамильтониан $H$ с нулевым следом.
    *   `GenerateLp`: Генерирует случайный оператор Линдблада $L$ с нулевым следом.

*   **`matrix_decomposition.h`**
    Реализует быстрые аналитические формулы для перехода между матричным представлением ($\rho$, $H$, $L$) и векторным представлением ($v$, $h$, $l$) в базисе генераторов $SU(N)$.
    *   `GetHCoef`, `GetLCoef`, `GetVCoef`: Раскладывают гамильтониан, оператор Линдблада и матрицу плотности по базису, возвращая векторы их коэффициентов. Сложность по времени и памяти $O(N^2)$.
    *   `GetDensityBySUDecomposition`: Выполняет обратную операцию — восстанавливает матрицу плотности $\rho$ из вектора коэффициентов $v$. Сложность по времени и памяти $O(N^2)$.

*   **`original.h`**
    Содержит базовые, математически прямые функции для работы с матрицами.
    *   `Commutator`, `AntiCommutator`: Высокопроизводительные реализации коммутатора и антикоммутатора с использованием функции `cblas_zgemm` из MKL.
    *   `CreateBasisArray`: Создает явное матричное представление (в памяти) для генераторов базиса $SU(N)$.

*   **`lindblad_utils.h`**
    Ключевой модуль, в котором вычисляются компоненты векторизованного уравнения ($Q$, $R$, $K$).
    *   `GenerateTensorF`, `GenerateTensorD`: Генерируют в разреженном формате COO (список координат) вещественные структурные тензоры $f_{mns}$ и $d_{mns}$, которые являются основой для построения матриц $Q$, $R$ и вектора $K$. Сложность по времени и памяти $O(N^3)$.
    *   `GenerateTensorZ`: На основе тензоров $f$ и $d$ генерирует в COO-формате тензор $z$. Сложность по времени и памяти $O(N^3)$.
    *   `GenerateCOOMatrixQ`: На основе тензора $f$ и коэффициентов гамильтониана $h$ строит матрицу $Q$ в COO-формате. Сложность по времени $O(N^3 log N)$, по памяти $O(N^3)$.
    *   `GenerateMatrixR`: На основе тензоров $f$, $z$ и коэффициентов операторов Линдблада $l$ строится матрица диссипации $R$. Сложность по времени $O(N^5)$, по памяти $O(N^4)$.
    *   `GenerateVectorK`: На основе тензора $f$ и коэффициентов операторов Линдблада $l$ строится вектор $K$. Сложность по времени $O(N^3)$, по памяти $O(N^2)$.

*   **`lindblad_utils_dimer.h`**
    Содержит реализацию для конкретной физической модели «димера», а также высокооптимизированные классы для построения разреженных матриц.
    *   `struct DimerModel`: Инкапсулирует физические параметры модели и методы для получения зависящих от времени коэффициентов гамильтониана.
    *   `struct SparseQBuilder`: Эффективно строит матрицу $Q$ напрямую в формате CSR (Compressed Sparse Row), готовом для использования в решателях MKL. За $O(N^3 log N)$ времени и $O(N^3)$ памяти создает алгоритмическую структуру, которая поддерживает *мгновенное обновление значений* (сложность $O(N^3)$ времени) при изменении гамильтониана без переаллокации памяти.
    *   `struct SparseRBuilder`: Аналогичный класс для однократного построения матрицы диссипации $R$ в формате CSR за $O(N^3 log N)$ времени и $O(N^2)$ памяти.

*   **`timer.h`**
    Вспомогательные утилиты для профилирования производительности.
    *   `class Timer`: Измеряет время выполнения участков кода (в микросекундах).
    *   `class RAMMeter`: Отслеживает потребление оперативной памяти (VSZ и RSS, только для ОС Linux).

### 🚀 Директория `/src` (Исполняемые файлы)

*   **`hardcore.cpp`**
    Реализует прямое решение уравнения Линдблада в его исходном матричном представлении, без перехода к векторизованной форме. Этот подход служит "точкой отсчета" (ground truth).
    *   **Представление данных:** Матрица плотности $\rho$, гамильтониан $H$ и операторы $L_p$ представлены в виде плотных матриц `MKL_Complex16*`.
    *   **Эволюция:** Временная эволюция вычисляется методом Рунге-Кутты 4-го порядка непосредственно для матрицы $\rho$.
    *   **Функция `LiouvillianOfRho`:** На каждом шаге РК4 вычисляет правую часть уравнения Линдблада, используя вызовы `cblas_zgemm` из MKL.
    *   **Роль векторизации:** Вектор состояния $v$ вычисляется функцией `SUBasisDecompose` только в конце шага для вывода и анализа. Сама динамика рассчитывается в матричном пространстве.

*   **`original.cpp`**
    «Лобовая» (наивная) реализация векторизованного подхода. Служит для демонстрации строгой математической основы метода без технических оптимизаций.
    *   **Явное создание базиса:** В памяти создается полный набор из $N^2-1$ базисных матриц (функция `CreateBasisArray`).
    *   **Вычисление через след:** Коэффициенты $h, l, v$ вычисляются по определению (например, $ v_{i} = \mathrm{Tr}(\rho F_{i}) $), что требует $O(N^4)$ операций.
    *   **Тензоры через коммутаторы:** $f_{mns}$ и $d_{mns}$ вычисляются путем явного перемножения матриц базиса и взятия следа. Производительность и потребление памяти в этом файле на порядки хуже, чем в `main.cpp` ($O(N^6 A)$ времени и $O(N^6)$ памяти, $A$ — сложность перемножения двух разреженных матриц).

*   **`main.cpp`**
    Основная, высокооптимизированная реализация, использующая векторизованный подход для систем с **не зависящим от времени** гамильтонианом.
    *   **Предварительные вычисления:** Вся тяжелая математика выполняется до старта времени:
        1. Начальные матрицы $\rho_0, H, L_p$ раскладываются по базису $SU(N)$ ($v_0, h, l_p$).
        2. Генерируются тензоры $f_{mns}$, $d_{mns}$, $z_{mns}$.
        3. Строятся матрицы унитарной эволюции $Q$ и диссипации $R$, а также вектор $K$.
        4. Формируется общая матрица эволюции $S = Q + R$.
    *   **Эволюция:** На каждом шаге РК4 решается система ОДУ $\frac{dv}{dt} = Sv + K$. Шаг сводится к одной высокооптимизированной операции «матрица-вектор» (`cblas_dgemv`).
    *   **Восстановление:** Вектор $v(t_{end})$ преобразуется обратно в матрицу плотности $\rho(t_{end})$.


*   **`main_dimer.cpp`**
    Оптимизированная реализация для модели димера с **зависящим от времени гамильтонианом** $H(t)$.
    *   **Использование «сборщиков»:** Матрицы $Q$ и $R$ строятся сразу в формате CSR с помощью `SparseQBuilder` и `SparseRBuilder`.
    *   **Эффективное обновление:** Так как от времени зависит только $H(t)$, меняется только матрица $Q(t)$. Ее структура разреженности остается постоянной. На каждом шаге (4 раза за шаг РК4) вызывается `q_builder->UpdateValues(*h_coef)`, что пересчитывает *только значения*, не перестраивая индексы матрицы.
    *   **Разреженная арифметика:** Умножение матрицы на вектор выполняется функцией MKL для разреженных матриц `mkl_sparse_d_mv`, что кардинально ускоряет расчет по сравнению с плотным `dgemv`.

*   **`main_dimer_slow.cpp`**
    Неоптимизированная версия решателя для модели димера (для проверки корректности и сравнения производительности).
    *   **Отсутствие «сборщиков»:** Используется базовая функция `GenerateCOOMatrixQ`.
    *   **Полный пересчет:** На каждом подшаге Рунге-Кутты происходит полный цикл с нуля: обновление $H(t)$ $\rightarrow$ разложение в $h(t)$ $\rightarrow$ *полное перестроение* матрицы $Q(t)$ из тензора $f$. Это приводит к огромным вычислительным издержкам.

*   **`tensor_to_file.cpp`**
    Вспомогательная утилита для экспорта сгенерированных структурных тензоров алгебры $SU(N)$ в обычные текстовые файлы.
    *   **Генерация и экспорт:** Программа вычисляет разреженный тензор $f/d/z$ и построчно записывает индексы $(m, n, s)$ всех его ненулевых элементов в `.txt` файл.
    *   **Интеграция и анализ:** Этот скрипт служит мостом для связи C++ кода с внешними аналитическими инструментами. Сгенерированные файлы координат можно легко загрузить в Python (NumPy/SciPy), MATLAB или Mathematica.
    *   **Назначение:** Идеально подходит для отладки, визуализации паттернов разреженности (sparsity patterns) матриц $Q$ и $R$, а также для независимой проверки структурных констант при увеличении размерности системы $N$.

## 🛠 Зависимости

*   **С++ Компилятор:** С поддержкой стандарта C++20 (GCC, Clang, MSVC, Intel C++ Compiler). Проект разрабатывался и запускался с использованием GCC13.
*   **CMake:** Версия 3.15 или выше.
*   **Intel MKL:** Библиотека должна быть установлена в системе. Проект по умолчанию настроен на поиск `MKLSequential` (последовательная версия), но может быть легко переключен на параллельную.

## ⚙️ Сборка и запуск в консоли

Проект использует стандартный out-of-source билд через CMake.


1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/dochkavurdalaka/LindbladEquationSolver/
    cd LindbladEquationSolver/
    ```

2.  **Создайте директорию для сборки:**
    ```bash
    cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
    ```
    -   `-S .`: указывает, где находятся исходные файлы (текущая директория).
    -   `-B build`: указывает, где будет создана директория сборки (`build`).
    -   `-DCMAKE_BUILD_TYPE=Release`: собирает проект в Release-режиме (с оптимизациями).

3.  **Скомпилируйте проект:**
    ```bash
    cmake --build build
    ```

После успешной сборки исполняемые файлы будут находиться в директории `build/bin/Release/`.
### Варианты сборки

-   **Debug-сборка** (с отладочной информацией и без оптимизаций):
    ```bash
    cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
    cmake --build build
    # Исполняемые файлы будут в build/bin/Debug/
    ```

-   **Сборка с санитайзером** (для поиска утечек памяти, неопределенного поведения и data-race-ситуаций):
    ```bash
    cmake -S . -B build -DCMAKE_BUILD_TYPE=Asan
    cmake --build build
    # Исполняемые файлы будут в build/bin/Asan/
    ```
